# 2026-06-29 工作日志

## 词云布局修复

**问题**：用户报告"词云乱了"。截图显示所有词都堆在上半区，下半区空白。

**根因**：
1. 旧算法用 `t = i/(N-1)` 把所有词按索引线性映射到内圈到外圈的半径，导致大权重的前几个词（i=0,1,2,3...）全部分到 `radius=0.15` 的内圈，造成中心大词全部堆叠。
2. CSS `.cloud-canvas` 设了 `max-height: 480px` 限制了高度，但算法按 `aspect-ratio: 4/3`（H=750）算坐标，导致下半部分的词位置超出画布。

**修复**：
- 重写 `renderKeywordCloud` 使用 **Archimedean 椭圆螺旋 + AABB 矩形碰撞检测**：
  - 最大的词放画布正中
  - 后续词从第 1 圈开始，按椭圆周长均匀采样角度，逐圈外扩
  - 矩形碰撞：用每个词的 `getBoundingClientRect()` 测真实像素尺寸，按旋转角计算外接矩形
- 画布坐标系改为 `W=1000, H=500`（接近 16:9），与 CSS `aspect-ratio: 16/9` 一致
- CSS `.cloud-canvas` 改为 `aspect-ratio: 16/9; max-height: 560px`

**结果**（验证）：
- 30 个词全部渲染，0 个被推到画布外
- 紧凑椭圆布局：中心"避坑"，次层"指南/教程/推荐/区别/怎么选/教你"，外层"打工人/女生/学生党/姐妹/朋友/新手/工作..."环绕
- 单词字号 19.2~48px，旋转 -35°~+35°，单一蓝色调（hsl 220 90% 35%~60%）
- 已部署到 CloudStudio + GitHub Pages（commit df23942）

**关键经验**：词云算法里 `aspect-ratio` 必须和 CSS 完全一致，否则坐标会溢出画布。
