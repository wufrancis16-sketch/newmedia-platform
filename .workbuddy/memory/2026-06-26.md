## 词云模块修复（重要）

**问题**：词云（关键词云 tab）一直显示固定内容，不随每日更新变化。

**根因**：
- `public/js/app.js` 的 `renderKeywordCloud` 函数中，`trendingKeywords` 数组是**硬编码**在源码里的固定内容（"2026高考"、"芒种"、"油价调整"等娱乐热点词）
- `update-data.js` 只更新 `REAL_DATA` 变量（小红书/抖音/公众号数据），**完全没碰词云数组**
- 词云既不来自接口，也不基于真实标题动态生成 → 永远不变

**修复方案**：
1. `app.js`：硬编码数组改为读取 `KEYWORD_CLOUD_DATA` 全局变量，注入失败时使用 `FINANCE_KEYWORDS_FALLBACK` 兜底
2. `update-data.js`：新增 `generateKeywordCloud(items)` 函数
   - 统计 `item.tag` 频次（tag 权重 ×3）
   - 从标题用 `/[\u4e00-\u9fa5]{2,4}/g` 提取 2-4 字词
   - 过滤停用词（常见助词 + 业务泛词"内容/教程/方法/技巧"等 + 包含停用词片段的复合词如"相关内容"）
   - 合并 + Top30 截取 + 不足用 FINANCE_KEYWORDS 兜底
   - 归一化权重到 10-100
3. `update-data.js` 注入到 `app.js`：`/const KEYWORD_CLOUD_DATA = \[[\s\S]*?\n\];/` 正则替换；首次运行则插入到 REAL_DATA 之后

**验证结果**（2026-06-26 16:26）：
- 抖音抓到 25 条真实视频 → 补充到 66 条
- 公众号抓到 0 条（搜狗反爬） → 补充到 67 条
- 词云注入 30 个关键词：ERP、财务软件、做账、进销存、财务管理、ERP系统、库存管理、业财一体、出纳、项目管理、开单软件、出入库...

**注意**：stowords 不要加"系统"、"管理"这类业务词，否则会误伤"ERP系统"、"库存管理"。
